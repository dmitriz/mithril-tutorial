Hyperscript

Introducing m(selector, attributes, children)

Let's render some actual HTML. This time, we'll use Mithril's hyperscript function m(), which lets us describe any HTML structure we want using pure JavaScript. The output is a vnode, which is an object intended to be a part of Mithril's virtual DOM tree.

So let's put it into practice! For now, Let's just try an h1, with a simple attributes object and a single text child:

m.render(document.body, m("h1", {title: "I'm big"}, "Hello! You're looking well."))

m() takes 3 arguments:

  1. selector (required): a CSS query string, such as:
    • "p" — a paragraph tag
    • "h2.subhead" — an h2 with class "subhead"
    • "div#hero" — a div with id "hero"
    • "input[type=password][placeholder=Password]" — you get the idea.
    It's considered good practice to set static attributes right here in the selector.
  2. attributes (optional): an object which contains keys for computed attributes/properties and event handlers for the vnode:
    • { onclick: someFunctionWithoutParentheses }
    • { oninput: function(){ myVar = someValue } }
    • { title: somethingIsTruthy() ? "Hey, that's true!" : "No, not even." }
    This gives us 2 places to set attributes.

    m("img[alt=foo]")

    has exactly the same result as

    m("img", {alt: "foo"}).

    It's considered good practice to set dynamic attributes in the attributes object, and leave statics in the selector.
  3. children (optional): an array or splat of child vnodes. 
    m("div", [
	m("p", "foo"), 
	m("p", "bar")
])
    ...has the same result as... 
    m("div"
	, m("p", "foo")
	, m("p", "bar")
)
    ...which is... 
    <div>
	<p>foo</p> 
	<p>bar</p>
</div>
