Components

The next step is to add some simple interactivity. Let's start by updating a button's label every time it's clicked. We'll need 2 new concepts: components and m.mount().

A Mithril component is just an object with a view function, which returns a vnode. Here, our vnode will just be a button.

In order for us to tap into Mithril's auto-redraw functionality, we'll need to abandon m.render() , replacing it with m.mount().

m.mount() is similar to m.render() in that it takes a target element as its first argument, but instead of then taking a vnode, it takes a component. Event handlers defined in a component's view trigger automatic redraws, so any data changes will be instantly reflected in the view. In this case, our data is just the integer variable count.

var count = 0

var MyComponent = {
	view: function(){
		return m("button"
			, { onclick: function() { count++ } }
			, "Count: " + count
		)
	}
}
	
m.mount(document.body, MyComponent)

Simple! We have a component MyComponent which returns a button vnode. Clicking our button increments the variable count. Since our button's text relies on the value of count, it too will be updated with each click.

Any event handler which is defined in a component's view will trigger a new diff, and if our data has changed, an automatic redraw.
