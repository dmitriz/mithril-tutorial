Jumping right in, let's grab Mithril from a CDN.

<html>
<body>
	<script src="//unpkg.com/mithril/mithril.js"></script>
	<script>

	// Your code goes here
		
	</script>
</body>
</html>

And now let's write our first Mithril SPA:

var MyComponent = {
	view: function() { 
		return m('h1', 'Hello, World!')  
	}
}
	
m.mount(document.body, MyComponent)

Ok, so what's going on here? Assuming you're not new not JavaScript, this shouldn't look too scary. There are 3 Mithril-specific constructs (links are to the official documentation):

  • Components
  • Hyperscript
  • m.mount()

In Mithril, a component is just an object with a view function. The view function must return a virtual DOM node, called a vnode. So here you can see the object MyComponent, which has a view function, which returns a call to a method called m(), which is the Mithril hyperscript utility that creates vnodes.

m(), also known as hyperscript, is Mithril's templating system. It allows us to express HTML structures using JavaScript syntax. It's very flexible and very powerful.

m.mount injects a component into an HTML element, and wires it to Mithril's auto-redraw apparatus, which is where quite a bit of the magic happens.
